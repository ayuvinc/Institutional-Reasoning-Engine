# Security Policy

## Supported Content

This repository is documentation-first (whitepaper + builder guide + hosted docs).

## Reporting a Vulnerability

If you discover a security issue related to published artifacts or site hosting:

1. Do not open a public issue with exploit details.
2. Report privately to the maintainer listed in `README.md`.
3. Include:
   - Description of the issue
   - Steps to reproduce
   - Affected file/path
   - Potential impact

## Response Targets

- Initial acknowledgment: within 5 business days
- Triage/update: within 10 business days

## Disclosure

Please allow time for review and remediation before public disclosure.
