# Contributing to Institutional Reasoning Engine

Thank you for improving this repository.

## Scope

This repository currently hosts:
- IRE v1.1 whitepaper artifacts
- Interactive builder guide
- Public docs site (`docs/`) for GitHub Pages

Contributions should improve clarity, accuracy, accessibility, and implementation-readiness.

## How to Contribute

1. Fork the repository.
2. Create a branch from `main`.
3. Make focused changes with clear commit messages.
4. Open a pull request using the PR template.

## Pull Request Expectations

- Keep PRs scoped to one area (docs, site UX, references, governance, etc.).
- Explain why the change is needed.
- Include before/after screenshots for visual site changes.
- Update `CHANGELOG.md` under `Unreleased`.

## Content Quality Rules

- Use plain language first; define technical terms.
- Do not introduce legal claims as legal advice.
- Cite sources when adding standards/regulation references.
- Keep terminology consistent: IRE, base components, clusters, deployment profiles.

## Commit Message Guideline

Use concise prefixes:
- `docs:` documentation updates
- `site:` docs site updates
- `meta:` repo/process/governance updates
- `fix:` corrections

Example:
`docs: clarify profile 4 activation criteria`

## Local Checks Before PR

- Confirm links in `README.md` and `docs/index.html` still work.
- Confirm GitHub Pages still renders from `/docs`.
- Confirm files open correctly (`PDF`, `DOCX`, `HTML`).
