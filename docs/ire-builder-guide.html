What Is an Institutional Reasoning Engine?

An IRE is not a chatbot. It is not an automation tool. It is an AI system that runs a structured investigation workflow — forming a hypothesis, iterating through evidence, and requiring a human to approve every finding before it enters the record.

The One-Sentence Version

An IRE is a professional investigation workflow encoded in software — AI accelerates the analysis, humans remain accountable for every conclusion.

The Three Things That Make It Different

🔒 Methodology as Engineering

Investigation rules are baked into the system — not left to human discretion. The AI cannot skip a step any more than a calculator can divide by zero.

👤 Human Gate at Every Finding

No AI output enters the official record without a named investigator approving it. The approval IS the record — not a step before the record.

🔗 Tamper-Evident Audit Chain

Every action is cryptographically linked. If anyone alters a log entry — even one character — every subsequent entry breaks. Mathematically detectable.

The 8 Mandatory Base Components

#ComponentWhat It Does (Plain English)Why Non-Negotiable
B1Case-Scoped IsolationEach case gets its own sealed container. Evidence from Case A cannot leak into Case B — by design.Cross-contamination would make findings legally indefensible.
B2Retrieval VerifierBefore any AI claim reaches an investigator, the system checks it exists in the actual case documents. Not AI judgment — a database lookup.Prevents hallucinated findings entering the record.
B3Evidence GrounderEvery verified claim gets a citation tag — document name, page number, chunk ID. Like footnotes, but automatic and tamper-evident.Regulators and courts demand traceable findings.
B4Human Review GateNo AI finding enters the official record without a named investigator approving it. The approval event is itself logged.Accountability. A human is responsible for every finding.
B5Audit Event LoggerEvery action logged: what was queried, what was retrieved, which AI model was used, who approved what, when.You cannot defend an investigation record you cannot reconstruct.
B6Hash-Chain ImmutabilityEach log entry is cryptographically linked to the previous one. Altering any entry breaks every entry after it.Proves the record was not tampered with after the fact.
B7Model Version PinningEvery case stores exactly which AI model version was used. Upgrades during a live investigation are blocked.Reproducibility. You must be able to re-run the same reasoning months later.
B8Algorithmic Bias MonitorMonitors whether the AI is systematically over-flagging any particular demographic group. Active from day one.Regulatory risk. Biased AI in regulated investigation is a liability, not a product quirk.

The 7 Modular Clusters

Cluster A

Documentary Evidence

Turns any document pile into a searchable evidence base. PDFs, spreadsheets, emails — all queryable by meaning.

Cluster B

Behavioural & Interview

Detects contradictions between interview statements and documentary evidence. Cites both sides automatically.

Cluster C

Entity & Network Intelligence

Maps who is connected to whom. Surfaces circular flows and fraud rings invisible from individual documents.

Cluster D

Advanced Reasoning

The recursive reasoning engine. Forms hypotheses, tests them, revises them, repeats until confident or flagged.

Cluster E

Privacy Gateway

Replaces real identifiers with tokens before anything reaches external AI. De-tokenises only on approval.

Cluster F

Institutional Memory

Fine-tunes the AI on your own completed cases. The system learns your fraud typologies over time.

Cluster G

Integrity Elevation

Daily external anchoring of the audit chain. Elevates from institutional-grade to court-grade evidence integrity.

Most Common Ingestion Failures

Scanned PDFs without OCR layer: Build a preprocessing check — if PyMuPDF returns no text, route to Tesseract automatically. Password-protected files: Require investigator to decrypt before upload — do not store passwords. Non-standard CSV encodings: Detect encoding with chardet before parsing.

Critical: Do Not Skip Entity Resolution

A 10% entity duplication rate in a 500-node graph produces 50 phantom nodes — enough to break circular flow detection entirely. This is the gap that most investigation AI systems fail to address honestly. Build Tier 2/3 resolution before building the graph layer.

Evidence Score Thresholds

0.80 — Section Lock

Individual report sections lock at this score. Still requires minimum 3 citations.

0.90 — Final Finding Lock

Final finding sections (preliminary findings, regulatory exposure) require this higher threshold.

High Score + Thin Evidence

If score is high but citations are fewer than 3, the system flags: "High confidence on thin evidence — manual review required." Do NOT auto-lock.

Known Limitation — Inference Re-Identification

Pseudonymisation controls what enters the AI — it does not control what the AI infers. A model given pseudonymised data may produce output that, combined with other information, re-identifies a data subject. This is why the Human Checkpoint exists — investigators must review for this risk before approving. Get legal counsel to review your pseudonymisation approach before going live.

Which Profile Should You Start With?

If you process documents: start at Profile 3. If your cases primarily involve interviews: start at Profile 2. If you need entity network mapping: go to Profile 4. If you are a regulated financial institution with external AI: Profile 5. Start at the lowest profile that covers your current needs — you can upgrade incrementally without rebuilding.
